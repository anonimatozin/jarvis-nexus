## MODULOS IMPLEMENTADOS

### 1. Self-Evolving Skills (modules/skills/)
- SkillEvolver: Aprende e armazena skills automaticamente
- Skills persistem em JSON (data/learned_skills.json)
- Comandos: "listar skills", "criar skill", "remover skill"

### 2. Proactive Autonomy (modules/autonomy/)
- ProactiveEngine: 4 niveis (OFF, MAINTENANCE, PROACTIVE, FULL_AUTO)
- Monitora inatividade e executa tarefas proativamente
- Comandos: "nivel autonomia [0-3]", "status autonomia", "iniciar manutencao"

### 3. Safety Tiers (modules/security/)
- SafetyTiers: 26 acoes mapeadas (SAFE, CAUTION, DANGER, CRITICAL)
- Senha mestra para acoes criticas
- Comandos: "status seguranca", "definir senha mestra", "verificar senha", "aprovar acao"

### 4. Goal Pursuit - OKR (modules/productivity/)
- GoalPursuit: Sistema de objetivos e resultados-chave
- Progresso de 0.0 a 1.0 por objetivo
- Comandos: "criar objetivo", "adicionar KR", "progresso do objetivo", "proximas acoes"

### 5. Dev Agent v3.0 (modules/dev_agent/)
- Ferramenta unica: OpenCode (sem OpenClaude)
- Hierarquia: OpenCode -> Brain (fallback)
- Comandos: "criar arquivo", "ler arquivo", "editar arquivo", "rodar script", "analisar codigo"

## HOSPEDAGEM GRATUITA - BOT DISCORD / SITE 24/7

### Oracle Cloud Always Free (Recomendado)
- RAM: 24GB ARM ou 1GB AMD
- Uptime: 99.9%
- Site: cloud.oracle.com
- Como: Criar conta, escolher "Always Free"
- Risco: Pode deletar instancias apos 60 dias sem aviso
- Ideal para: Bots pesados, scrapers, projetos grandes

### Kerit Cloud (Mais facil)
- Gratuito, sem cartao de credito
- Suporta: Python, Node.js
- Painel: Pterodactyl (facil de usar)
- Site: kerit.cloud
- Ideal para: Iniciantes, bots simples

### Bot-Hosting.net
- Gratuito: 256MB RAM, 20% CPU
- Renovacao: a cada 4 dias (manual)
- Suporta: Python, Node.js, Java, Deno
- Site: bot-hosting.net
- Ideal para: Bots pequenos

### Google Cloud Free Tier
- Instancia: e2-micro
- Uptime: 99.5%
- Ideal para: Bots simples, APIs

### PythonAnywhere
- RAM: 512MB
- Uptime: 100% (nativo)
- Ideal para: Scripts simples, APIs basicas

### Fly.io
- Gratis com limite de 3 apps
- Suporta: Python, Node.js, Docker
- Deploy via Git ou Docker
- Ideal para: Bots com Docker

### Render.com
- Gratis com limitacoes
- Web services dormem apos 15 min
- Background workers pagos
- Nao recomendado para bots 24/7

### Replit
- Gratis com limitacoes
- Dorme apos 5 min de inatividade
- Bloqueou workarounds (UptimeRobot)
- Nao recomendado para 24/7

## CORRECOES DE AUDIO (2026-07-09)

### Problemas encontrados e corrigidos:
1. Erro de encoding no logger.py (banner ASCII art)
   - Adicionado file=open com encoding='utf-8'
2. Device de audio oficial truncado no hud_settings.json
   - "Speakers (High Definition Audio" causava No such device
   - Campo audio_output_official_encrypted limpo
3. edge-tts erro 403 (versao 6.1.12)
   - Atualizado para edge-tts 7.2.8
4. Verificacao de device inexistente no tts_engine.py
   - Adicionada verificacao via sounddevice.query_devices()
5. Metodo speak() faltando no hybrid_voice.py
   - Engine chamava self.tts.speak() mas HybridVoice so tinha falar()
   - Adicionado metodo speak() que delega para falar()

## PROXIMOS PASSOS - DEPLOY GITHUB + NETFLIFY

### Passo 1: Login GitHub
```powershell
$env:Path = [System.Environment]::GetEnvironmentVariable("Path","Machine") + ";" + [System.Environment]::GetEnvironmentVariable("Path","User")
gh auth login --web -p https
```
- Abrir link no browser
- Copiar codigo e colar

### Passo 2: Criar repo e push
```powershell
cd C:\Users\Administrator\Desktop\JARVIS
git init
git add .
git commit -m "JARVIS Control Center v2.0"
gh repo create jarvis-nexus --public --source=. --remote=origin --push
```

### Passo 3: Netlify
1. Entrar em app.netlify.com
2. "Add new site" → "Import an existing project"
3. Escolher GitHub
4. Selecionar repo "jarvis-nexus"
5. Build settings:
   - Build command: `pip install -r netlify/functions/requirements.txt`
   - Publish directory: `dashboard/static`
   - Functions directory: `netlify/functions`
6. Deploy!

### Deploy automatico:
- Toda vez que fizer push pro GitHub, o Netlify atualiza sozinho

### URL do site:
- Vai ficar algo como: https://jarvis-nexus.netlify.app

### IMPORTANTE:
- O .env NAO sobe pro GitHub (protegido pelo .gitignore)
- No Netlify, adicionar variaveis de ambiente manualmente:
  - GROQ_API_KEY
  - GEMINI_API_KEY
  - DISCORD_TOKEN
  - etc.
