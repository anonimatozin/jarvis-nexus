## UPGRADE RECENTE (2026-07-10)

### 🧠 UNIVERSAL KNOWLEDGE SYSTEM
**Localização:** `core/universal_knowledge/`

Sistema de conhecimento universal com lazy activation. Módulos ficam "dormindo" e acordam sob demanda.

**Arquitetura:** Híbrida (Factory Pattern + SQLite + Keywords/Semântica)

**Componentes:**
- `base_module.py` - Classe base (estados: SLEEPING, LOADING, ACTIVE, ERROR)
- `module_factory.py` - Factory Pattern com Lazy Loading, cleanup automático após 5min
- `knowledge_base.py` - SQLite (knowledge, module_cache, action_log)
- `intent_detector.py` - Detecção híbrida: 70% keywords + 30% semântica
- `universal_knowledge.py` - Orquestrador principal com segurança integrada

**11 Módulos Implementados:**

| Módulo | Descrição | Status |
|--------|-----------|--------|
| gmail | Emails via Gmail API | Precisa OAuth2 |

### 🔒 SISTEMA DE SEGURANÇA
**Localização:** `core/security/`

Proteção contra prompt injection, comandos perigosos e acesso não autorizado.

**Componentes:**
- `input_sanitizer.py` - Sanitização + Detecção de Injection + Permissões
- `sandbox.py` - Isolamento de comandos + Controle de arquivos + Segurança de rede
- `audit_logger.py` - Log completo de ações e eventos de segurança

**Proteções:**

| Proteção | Descrição |
|----------|-----------|
| Input Sanitization | 10+ padrões perigosos detectados |
| Prompt Injection | 25 frases suspeitas + 8 padrões de role |
| Command Blocking | curl, wget, bash, eval, python -e, etc. |
| Permission Manager | 7 ações exigem confirmação |
| Command Sandbox | Isolamento de comandos |
| File Access Control | Protege .env, credentials, keys |
| Network Security | HTTPS-only, domínios bloqueados |
| Audit Log | SQLite com 3 tabelas |

**Ações que exigem confirmação:**
1. send_email
2. delete_file
3. execute_command
4. access_credentials
5. modify_system
6. install_software
7. change_settings

**Níveis de ameaça:**
- SAFE (0) - Normal
- LOW (1) - Informativo
- MEDIUM (2) - Revisar
- HIGH (3) - Bloqueado
- CRITICAL (4) - Ameaça crítica
